INNER JOIN

  • Vous devez maintenant savoir que JOIN n’est en fait qu’une méthode de jointure parmi d’autres
  • C’est la plus courante et elle est donc toujours appliquée par défaut lorsque vous écrivez le mot clé JOIN dans votre instruction SQL
  • Techniquement parlant, cependant, son nom complet est INNER JOIN.

INNER JOIN

Maintenant, utilisez le nom complet INNER JOIN pour relier les tables chambre et equipement, de sorte que chaque équipement soit affiché avec sa chambre et d’autres colonnes pertinentes

INNER JOIN

Le résultat doit comporter les colonnes suivantes :

  • chambre_id - identifiant de la chambre
  • no_chambre
  • nb_lits
  • no_etage
  • equipement_id - identifiant de l’équipement
  • nom (de l’équipement)

INNER JOIN

SELECT chambre.id, no_chambre, nb_lits,
no_etage, equipement.id, nom
FROM chambre
INNER JOIN equipement
ON equipement.chambre_id = chambre.id;

Comment fonctionne une jointure interne ?

  • Si vous comparez maintenant les résultats de INNER JOIN avec le contenu de la table equipement, vous remarquerez que tous les équipements ne sont pas présents dans la table résultante
  • Par exemple, une belle bouilloire avec l’identifiant 11 n’y figure pas
  • Savez-vous pourquoi ?

Comment fonctionne une jointure interne ?

  • INNER JOIN (ou JOIN, en abrégé) n’affiche que les lignes des deux tables pour lesquelles il existe une correspondance entre les colonnes
  • En d’autres termes, vous ne pouvez voir que les équipements auxquels une chambre a été attribuée et vice versa
  • Les équipements qui ne sont attribués à aucun local n’apparaissent pas dans le résultat

LEFT JOIN

LEFT JOIN fonctionne de la manière suivante :

  • il renvoie toutes les lignes de la table de gauche (la première table de la requête)
  • plus toutes les lignes correspondantes de la table de droite (la deuxième table de la requête)

Exemple de LEFT JOIN

Imaginons 2 tables : Voiture et Personne

FROM voiture
LEFT JOIN personne
ON voiture.proprietaire_id = personne.id;

RIGHT JOIN

Le fonctionnement de RIGHT JOIN est le suivant :

  • il renvoie toutes les lignes de la table de droite (la deuxième table de la requête)
  • plus toutes les lignes correspondantes de la table de gauche (la première table de la requête).

RIGHT JOIN

Reprenons les 2 tables : Voiture et Personne

SELECT *
FROM voiture
RIGHT JOIN personne
  ON voiture.proprietaire_id = personne.id;

FULL JOIN

  • Le FULL JOIN n’existe pas avec MySQL
  • Mais existe pour les SGBDR suivants : PostgreSQL, Microsoft SQL Server, Oracle, etc
  • Donc c’est important de comprendre ce que c’est !

FULL JOIN

Le fonctionnement de FULL JOIN est le suivant :

  • il renvoie toutes les lignes des deux tables et combine les lignes lorsqu’il y a une correspondance
  • En d’autres termes, FULL JOIN est une union de LEFT JOIN et RIGHT JOIN

FULL JOIN

Reprenons les 2 tables : Voiture et Personne

SELECT *
FROM voiture
FULL JOIN personne
ON voiture.proprietaire_id = personne.id;

UNION

SELECT * FROM chambre
RIGHT JOIN etudiant

Le mot-clé OUTER

  • Vous vous souvenez que JOIN était le diminutif de INNER JOIN n’est ce pas ?
  • Les trois jointures que nous venons de mentionner : LEFT JOIN, RIGHT JOIN, et FULL JOIN sont également des raccourcis
  • Leurs noms complets sont OUTER JOIN: LEFT OUTER JOIN, RIGHT OUTER JOIN, et FULL OUTER JOIN
  • Vous pouvez ajouter le mot-clé OUTER et les résultats de vos requêtes resteront les mêmes

Le mot-clé OUTER

  • Vérifiez par vous-même
  • Utilisez le nom complet RIGHT OUTER JOIN pour afficher toutes les bouilloires avec les données de leur chambre (même si aucune chambre n’est attribuée)
SELECT *
FROM chambre
RIGHT OUTER JOIN equipement
ON equipement.chambre_id = chambre.id
WHERE nom = 'bouilloire';
